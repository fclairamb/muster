# 10 — End-to-end test harness

## Goal
A single Go test that exercises the whole stack — config, registry, repo
detection, org-prefix, state files, watcher, dispatcher, notifications,
TUI rendering, tmux session lifecycle — using only fakes for the things
that need a human (real Claude, real notifications) and real
implementations for everything else.

This is the "I can test the app myself" guarantee.

## Deliverables

### `test/e2e/e2e_test.go` (build tags: `e2e,tmux`)

Single test function. Skips if `tmux` is missing.

### Setup

1. `t.TempDir()` for `XDG_CONFIG_HOME`, `HOME`.
2. Build `bin/ssf` once via `go build`.
3. Create a fake claude script in tempdir:
   ```sh
   #!/bin/sh
   exec sleep infinity
   ```
   Set `SSF_CLAUDE_BINARY` to its path.
4. Create a fake git repo:
   ```
   git init /tmp/.../fake-repo
   cd fake-repo
   git remote add upstream git@github.com:stonal-tech/datalake.git
   git commit --allow-empty -m init
   ```
5. Inject a `RecordingNotifier` via a test-only flag the binary recognizes
   (`SSF_NOTIFIER=record:/tmp/.../notif.log`).

### Steps

1. Run `ssf <fake-repo>` via `os/exec`. Assert it exits 0 and the
   registry now contains the entry.
2. Start the TUI in-process via teatest (importing the same `tui`
   package the binary uses):
   - Initial render → entry shown as `s/datalake [main]` with no status
     (white-or-none).
3. Simulate a Claude state transition by writing
   `<repo>/.ssf/state/<slug>.json` with `kind=ready`. (We bypass the
   actual hooks here — that path is exercised by slice 05's manual smoke
   test, not this one.)
4. Wait up to 1s for the watcher → dispatcher → render pipeline to
   propagate. teatest snapshot now shows the row in green.
5. Assert the recording notifier file contains exactly one line:
   `Ready / ssf · s/datalake [main]`.
6. Drive `Enter` on the entry. teatest intercepts `tea.ExecProcess` and
   records the tmux command. Verify a real session named `ssf-<slug>`
   exists via `tmux -L ssf has-session`.
7. Drive `r`, confirm `y`. Assert:
   - The tmux session is gone.
   - The registry no longer contains the entry.
   - `.ssf/state/<slug>.json` is removed.
   - `.claude/settings.json` no longer references the slug.
8. Drive `q` to quit.

### Cleanup

- `t.Cleanup` kills any leftover tmux sessions on the `ssf` socket.
- `tmux -L ssf kill-server` is safe even if no server is running.

## Acceptance

```
make test-e2e
```

Green. Total runtime under 10 seconds.

## Notes

- This test is the safety net for refactors. Anything that breaks the
  full pipeline trips it.
- It does NOT cover real Claude Code hooks firing — that is slice 05's
  manual smoke test, intentionally separated because it requires a human.
- If this test ever needs a `time.Sleep` longer than 100ms, that's a
  signal something is missing a synchronization primitive — fix the code,
  not the test.
